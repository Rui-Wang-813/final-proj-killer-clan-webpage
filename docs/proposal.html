Final Project Proposal: Font Rendering

Rui Wang, Zhuowen Chen, Yifan Zhong, Yicheng Sun



Summary

We are going to build a Bezier curve rasterizer to render different TrueType .ttf or OpenType files. The output will be a .png file.

Problem Description

  1. We want to solve the problem of turning a .ttf file, and turning it into a .png image, with given or default texts.
  2. The first challenge we will be faced with is what is a TrueType font, what it is made of, and what elements inside it can be used to generate a Bezier curve, and eventually be rasterized. We plan to solve this problem by surfing the Internet for more details about the .ttf files.And here we found a library called FreeType, which can read in a TrueType file and give some primitives that we need.
  3. The second challenge is how we can rasterize a Bezier curve that we generated. We plan to solve this challenge by mixing the codes from Project 1 and 2.

Goals and Deliverables

The above is the built-in font viewer in Windows. We want to be able to take in an input of a .ttf file, and turn it into a .png image, with the similar effect to the font viewer. That being said, we want to output an image with a line of text with different characters, and different font size. The rendered image will be in .png format.

We plan to also offer an interactive system so that we can change the character we show, the size of the rendered image, and so on. In the demo, we will show every single feature of our interactive system.

We believe it is difficult to precisely evaluate the accuracy of font rendering. But we will try to compare between our rendered font and the results commercial font viwer to measure our code. Also, we will try to make our rasterization faster, which can be precisely measured by counting the time we take to render.

If we successfully achieve our baseline of the project, we will try to create a GPU based text rendering system.
